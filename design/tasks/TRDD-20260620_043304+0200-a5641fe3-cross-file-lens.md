## Problem

CAA's engine is map → filter → reduce, all per-file. A defect that exists only
in the **relationship between two files** is invisible to an agent that sees one
file. The PR path solves this with the `cross-layer` holistic lens; the non-PR
scan path does not have it, so `/caa-scan`, `/caa-audit-codebase`, and precommit
are blind to cross-file drift. The lens experiment measured this: 28/30 recall,
and the single MAJOR miss (GT11) was a cross-file env-var drift.

## Design (Option B — holistic once-per-run cross-file lens in scan mode)

Mirror the PR `XLAYER_PREFIX` for the non-PR review path:

- **Gate:** `TASK === 'review' && LENS !== 'pr' && wantXFile`, where
  `wantXFile = A.crossFile === true || (A.crossFile !== false && FILES.length >= 2)`.
  (PR mode already runs `cross-layer`; never double it. Single-file scopes skip
  by default but can force with `crossFile: true`; any scope can disable with
  `crossFile: false`.)
- **One agent, once per run** (phase `Cross-file`, label `lens:cross-file`),
  anchored on the audited `FILES` set, instructed to grep `ROOT` for the other
  end of each mismatch and to cite ≥2 different files per finding. Writes
  `TMP/__xfile.md`.
- **Resilience:** reuse the CGCP unbounded-RL sleeper-backoff. Hoist the PR
  block's `runLens` to module scope as `runResilientLens(prefix, label,
  expectName, phaseName)` — single source of truth, used by BOTH the new lens
  and the three PR lenses (a strict de-dup, no behavior change).
- **Reduce wiring:** add the cross-file report path as an input line to the
  non-PR `else` reduce, with the same "if unavailable, note it explicitly; never
  silently omit" rule the PR reduce uses.
- **Result object:** add `crossFileLens: <status>`.

### Why not Option A (per-file sibling injection)

Giving each per-file agent its "related" siblings requires knowing which files
are related. With no import graph in an arbitrary target repo (no `.codegraph`),
selection degrades to heuristics that are both fragile (wrong siblings) and
quadratic (every file × N siblings → token blow-up). The holistic lens lets ONE
agent do the cross-referencing by targeted grep — cheaper and proven. Option A
is deferred; if measurement later shows the holistic lens misses a class only
sibling-context catches, revisit.
